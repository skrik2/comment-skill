# comment-skill

**Comment SKILL** 是一套专为 AI Agent 设计的**评论分析与自动化处理SKILL**。

在后现代的赛博空间中，非建设性的辩经、情绪化宣泄和逻辑谬误充斥四周。

本 SKILL 的核心目的**不是为了证明自己绝对正确，更不是为了教导他人如何评论，而是为了帮用户过滤杂音、拒绝内耗，在纷繁的赛博空间里保持清醒与判断力。**

---

# 使用提示

用户需要自行设计 Agent 与社交平台交互的接口以及通知方法

# 核心工作流 (Workflow)

```
[接收评论输入] 
      │
      ▼
[防套娃约束检查] ──(触发套娃/驳斥原则)──► [ignore / delete] 熔断
      │
(通过检查)
      ▼
[规则模式匹配 (#1 ~ #10)]
      │
      ▼
[执行单一 Action] ──(若存在多个 Action 可选)──► 提示用户手动选择 / 无通知渠道则默认 ignore

```
# Actions

Agent 接收到评论后，会将其收敛判定为以下 **5 种标准动作**之一，拒绝多余的幻觉与过度发挥：

* `ignore`：忽略评论，不做任何回应（最常用的防内耗武器）。
* `delete`：删除评论（在知乎等平台自动执行“折叠”）。
* `block`：直接拉黑评论者。
* `notify`：通过指定渠道（Email/SMS/IM）向用户推送高价值或需注意的分析报告。
* `reply`：自动回复评论（如：请求澄清歧义或要求支付认知成本）。

# 📋 规则判定矩阵速查

本 SKILL 将复杂的互联网评论精准划分为以下维度并匹配了刚性动作：

| 规则章节 | 触发场景 | Agent 执行动作 | 核心逻辑说明 |
| --- | --- | --- | --- |
| **#0 认知防线** | 用户自身出现事实/逻辑错误 | **`notify`** | 提醒用户自身局限，提供正确源头。 |
| **#1 事实维度** | 评论者事实错误 / 观点混淆 / 身边统计学 | **`ignore`** | 缺乏可验证性或不影响大局的错误，直接忽略。 |
| **#2 情绪/攻击** | 纯情绪宣泄 / 人身攻击 / 评论后拉黑 / 听床 | **`ignore` / `delete` / `block`** | 净化评论区，反复违规者直接拉黑。 |
| **#3 错位维度** | 前提不同 / 定义差异 / 逻辑链错位 / 真空球形鸡 | **`ignore`** | 属于 Context Mismatch，不直接比较，不参与辩经。 |
| **#4 脑补攻击** | 稻草人谬误 / 错误归因 / 恶意解读 | **`ignore` / `delete`** | 汉隆剃刀优先；恶意解读直接删除。 |
| **#5.1 认知白嫖** | 伸手党 / 强行索要复杂解答 | **`reply` ➔ `delete`/`notify`** | 触发“v我50”机制，视对方后续支付行为动态调整。 |
| **#7 框架错位** | 降维/跨维度追问“为什么” | **`notify` / `ignore`** | 判断双方认知边界。未互动过则直接忽略。 |
| **#8 诉诸人身** | 间接/直接攻击身份或动机（含 *Whataboutism*） | **`ignore` / `delete`** | 视动机而定：意在比烂则忽略，意在深化则纳入 **#10**。 |
| **#9 ~ #10** | 存在歧义 / 带来新变量的建设性评论 | **`reply` / `notify`** | **节能模式下自动切换为 `ignore**`；正常模式下积极吸收。 |
